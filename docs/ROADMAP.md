# StudyPack — תוכנית עבודה לשיפור, ייעול ומיקצוע

מטרה: להפוך את StudyPack ממיזם אישי ל**מוצר אמיתי לסטודנטים** (אופק ארוך).
המסמך מסכם אודיט מלא לפי שבע עדשות — נגישות, עיצוב, קוד/אבטחה, ביצועים, ניסוח, מובייל ו-deploy — וממפה תוכנית עבודה מתועדפת בשלבים.

התאריך: 2026-07-23 · מבחן קרוב: 2026-07-28

---

## תקציר מנהלים

האפליקציה בנויה טוב בבסיסה: Next.js App Router עם רינדור בצד שרת, ולידציית Zod על כל ה-API, קריאות LLM עם "forced tool call" (בלי פרסום פרוזה), חסמי גודל להעלאות, security headers, ו-20 קבצי בדיקות. העיצוב הנאו-ברוטליסטי החדש נראה מצוין בדף הבית.

אבל כדי שסטודנטים אמיתיים ישתמשו בזה, יש **שלושה חוסמי השקה** שחייבים תיקון לפני הכל:

1. 🔴 **אובדן מידע ודאי בפרודקשן** — כל ערכה שמשתמש מייבא נעלמת (בעיית persistence).
2. 🔴 **אין הפרדת משתמשים אמיתית** — בלי Clerk כולם "אותו משתמש" ורואים/מוחקים אחד לשני.
3. 🔴 **חורי הרשאה** — משתמש יכול לתרגל מול ערכות של אחרים; אפשר "לחטוף" כיתה.

לצד אלה יש שכבה גדולה של ליטוש מקצועי (נגישות, מובייל, ניסוח, עקביות עיצובית) שתעשה את ההבדל בין "פרויקט" ל"מוצר".

**מפת הדרכים בקצרה:**

| שלב | מיקוד | משך משוער |
|---|---|---|
| **P0** | חוסמי השקה — מידע, אבטחה, ונגישות קריטית | ~1–2 שבועות |
| **P1** | יסודות מקצועיים — CI, הקשחה, עקביות עיצובית, ניסוח | ~3–4 שבועות |
| **P2** | מובייל, ביצועים, מדידה ואמינות | ~3–4 שבועות |
| **P3** | צמיחת מוצר — onboarding, תשלום, כיתות, PWA | מתמשך |

## ניצחונות מהירים (השבוע — לפני המבחן)

פעולות בטוחות, זולות, בעלות השפעה גבוהה — לא נוגעות ב-backend ומשפרות מיד את החוויה שלך ושל כל מי שרואה את האתר:

1. **ניגודיות כפתורים** — להחליף טקסט לבן על כתום → טקסט שחור (ink). כרגע ~2.5:1, נכשל בנגישות. (`globals.css` `.nb-btn-orange`)
2. **להשחיר את `--color-muted`** מ-`#6c6a5d` ל-~`#5a584c` — טקסט אפור קטן על הקרם כרגע לא עובר סף קריאוּת.
3. **לוקליזציה של דליפות אנגלית** — "Class created." / "Joined." / "Network error" / "Upload failed" / "CoursePack" → עברית.
4. **שם מוצר אחיד** — "StudyPack" בכותרת הדפדפן מול "סטאדיפאק" באתר; ומונח אחד לערכה ("ערכת לימוד") במקום ערכה/חבילה/CoursePack.
5. **תג "פעיל" חסר משמעות** — מופיע על כל כרטיס תמיד; להסיר או להפוך לסטטוס אמיתי.
6. **הגנת מקלדת** — ה-listeners של הכרטיסיות/מצגת חוטפים רווח/חצים גם בתוך שדות טקסט; להוסיף guard.

כל אלה ביחד הם ~חצי יום עבודה ומעלים משמעותית את רמת הגימור.

## P0 — חוסמי השקה (חובה לפני שסטודנטים נכנסים)

### 0.1 🔴 Persistence אמיתי — אחרת כל ייבוא נמחק
`vercel.json` מגדיר `STUDYPACK_STORE: memory`, כך שכל ערכה שנוצרת נשמרת ב-Map בזיכרון של אינסטנס serverless בודד — ונעלמת ברגע שהבקשה הבאה נוחתת על אינסטנס אחר. גם adapter ה-`fs` לא יעבוד: המערכת קבצים ב-Vercel היא ephemeral/read-only.
**פעולה:** לממש adapter עמיד (Vercel KV / Upstash Redis / Postgres / R2) מאחורי ה-`PackStore` interface הקיים (הקוד כבר בנוי לזה), ולהסיר את `memory` מ-`vercel.json`. להוסיף כתיבה אטומית ומכסות למשתמש.
**קבצים:** `src/lib/coursepack/store.ts`, `src/lib/classes/store.ts`, `vercel.json`. **מאמץ:** M.

### 0.2 🔴 לחייב Clerk בפרודקשן — לסגור את "demo-user-dev"
בלי מפתח Clerk, כל המבקרים הם משתמש קבוע אחד: משתפים ערכות, מוחקים אחד לשני, רואים כיתות זרות.
**פעולה:** ב-production, אם Clerk לא מוגדר → fail closed (לזרוק שגיאה, לא ליפול ל-dev-user). להפוך את ההתחברות לחובה על הראוטים המוגנים.
**קבצים:** `src/lib/auth/userId.ts`, `src/middleware.ts`. **מאמץ:** S–M.

### 0.3 🔴 חורי הרשאה — feedback וכיתות
- `POST /api/feedback` פותר ערכה לפי `packId` חופשי; אין אישור בעלות — משתמש יכול להריץ קריאות מודל (בתשלום) מול ערכות של אחרים. **פעולה:** לפתור ערכה רק מתוך ה-store של המשתמש עצמו או הרשאת-כיתה מפורשת; אחרת 404.
- `POST /api/classes` דורס `data/classes/{id}.json` בלי בדיקה — אפשר "לחטוף" מזהה כיתה קיים. **פעולה:** לבדוק קיום ולהחזיר 409 אם `ownerId` שונה (בדיוק כמו שכבר קיים ב-ingest).
**קבצים:** `src/app/api/feedback/route.ts`, `src/lib/coursepack/registry.ts`, `src/app/api/classes/route.ts`. **מאמץ:** M.

### 0.4 🔴 דף הנוסחאות לא שמיש במובייל
`.cs-page` ברוחב `210mm` עם `column-count:3` בפונט 8.4px — במסך 375px זה בלתי קריא ונוסחאות (`Ltr` mono) גולשות אופקית. זה נכס ליבה של המוצר.
**פעולה:** מתחת ל-640px — עמודה אחת, פונט ~13px, עמוד full-bleed בלי רוחב mm, ו-`overflow-x:auto` על קופסאות נוסחה.
**קבצים:** `src/app/globals.css` (בלוק `.cheatsheet`/`.cs-page`). **מאמץ:** S.

### 0.5 🔴 ניגודיות נגישות קריטית
- לבן על כתום (`.nb-btn-orange`) ≈ 2.5:1 — נכשל. → טקסט ink.
- `text-muted` על קרם ≈ 3.9:1 בטקסט קטן — נכשל. → להשחיר או להעלות גודל מינימלי.
- טקסט אדום מלא בקאלאאוטים (`text-mistake` על `bg-mistake-bg`) ≈ 4.4:1 — על הגבול. → גוף הטקסט ב-ink, אדום רק לתווית.
**קבצים:** `globals.css`, `PracticeView.tsx`, `FlashcardsView.tsx`, `ClassesView.tsx`, `CheatSheetView.tsx`. **מאמץ:** S.

## P1 — יסודות מקצועיים

### 1.1 CI/CD ואיכות בנייה
אין `.github/workflows`. שום דבר לא חוסם בנייה שבורה או טסט נכשל מלהגיע לפרודקשן.
**פעולה:** GitHub Action שמריץ `lint + test + build` על כל PR; לנעוץ Node (`"engines": { "node": ">=20" }` + `.nvmrc`).
**מאמץ:** S.

### 1.2 הקשחת עלות ואמינות ב-LLM
- Rate limiting כרגע process-local — נשבר על serverless (המכסה בפועל = capacity × מספר אינסטנסים). → Redis (Upstash) מבוזר.
- אין תקרת עלות per-user. → מכסת tokens/spend יומית ב-store העמיד.
- ה-API מחזיר `err.message` גולמי ללקוח (דליפת מידע). → לוג בצד שרת + הודעה גנרית + request id.
- אין `timeout`/`maxRetries` מפורשים ל-Anthropic SDK — ייבוא בודד יכול לרוץ ~300s ולהגיע ל-4 קריאות מודל. → להגדיר מפורשות.
- **Prompt-injection:** טקסט לא-מהימן ממסמכים נכנס למודל בלי בידוד. → לעטוף בתוחמים מפורשים ("התוכן הבא הוא נתונים, לא הוראות").
**קבצים:** `src/lib/ingest/rateLimit.ts`, `structure.ts`, כל ה-routes. **מאמץ:** M.

### 1.3 ייבוא כתהליך רקע עם התקדמות
כרגע `IngestForm` עושה `fetch` בודד חוסם עד 300 שניות; כל ניתוק רשת מאבד את כל הריצה, והמשתמש רואה ספינר בלי התקדמות.
**פעולה:** מודל job — enqueue → poll/SSE לסטטוס. מנתק את העבודה בתשלום ממשך חיי הבקשה.
**קבצים:** `IngestForm.tsx`, route חדש. **מאמץ:** L.

### 1.4 איחוד מערכת העיצוב + תיעוד
ההעברה לנאו-ברוטליזם חלקית. רכיבים שלמים נשארו בסגנון הישן, וכמה מכילים hex קשיח ששובר dark mode. אין תיעוד design-system — וזו הסיבה שהעיצוב "נזל".
**סדר עדיפויות להעברה:**
1. `AuthHeaderControls.tsx` — בכותרת הגלובלית, גלוי מאוד (ניצחון מהיר).
2. `PlanView.tsx` — נכס ליבה, כולו בסגנון ישן.
3. `VerifyView.tsx` — סגנון ישן + הכי הרבה hex קשיח.
4. `IngestForm.tsx` — זרימת היצירה המרכזית + פאנל בהיר שנשבר ב-dark.
5. `ClassesView.tsx`, `not-found.tsx`, `join/[code]`, ה-loaders/`Skeleton.tsx`, ה-toolbar של CheatSheet.
6. לתקן ב-`globals.css` את `#fff`/`#17160f`/`#f1eee4` הקשיחים ב-`.nb-hero-dark`/`.nb-pill-blue`/`.nb-btn-*` → tokens (תיקון ניגודיות ב-dark).
7. **לכתוב `docs/DESIGN-SYSTEM.md`** — טבלת tokens, כל `.nb-*` עם דוגמה, סקאלת radius/shadow, חוזה dark-mode, וחריג ההדפסה.
**חריג מוצהר:** רכיבי ה-cheatsheet להדפסה (`TopicBox/FormulaBox/Callout/TermKeyStrip/Summaries/.cs-box`) נשארים בסגנון צפוף להדפסה — לתעד כחריג מכוון. **מאמץ:** M–L.

### 1.5 ניסוח (UX Copy)
- לוקליזציה מלאה של הודעות סטטוס/שגיאה לעברית.
- שם מוצר ומונח-ליבה אחידים ("ערכת לימוד").
- להסיר ז'רגון שנחשף לסטודנטים: "rubric", "practiceItems", "CoursePack".
- לתקן "ש׳" (שניות → "שנ׳") ואת עקביות המגדר של "Claude".
- ריקון-מצב ידידותי בתרגול במקום הסבר טכני; תגית "פעיל" משמעותית או הסרה.
**קבצים:** `ClassesView.tsx`, `PracticeView.tsx`, `IngestForm.tsx`, `page.tsx`, `labels.ts`. **מאמץ:** S–M.

### 1.6 מנת נגישות
- שגיאות טפסים: `role="alert"`/`aria-live` + `aria-describedby`.
- Guard ל-key handlers גלובליים כשהפוקוס בשדה קלט.
- כפתורי דירוג מוסתרים → `disabled={!flipped}` (לא tabbable כשמוסתרים).
- אמוג'י/כוכבים: `aria-hidden` או label בעברית (כרגע "star"/"priority N" באנגלית).
- צבע ה-focus outline כתום-על-כתום → ink/navy.
- `aria-live` על פאנל המשוב וההתקדמות.
**מאמץ:** M.

## P2 — מובייל, ביצועים, מדידה

### 2.1 מובייל / רספונסיביות
- **נאבגשן הבית** צפוף ב-375px — להסתיר את תג ה-streak (כפילות עם הכרטיס הכהה), לכווץ פעולות משניות לתפריט.
- **toolbar המצגת** (~11 כפתורים) גולש ודוחף את השקופית מהמסך — במובייל להשאיר back/overview/fullscreen/next-prev, השאר ל-"⋯ עוד".
- מספרי ספירה תלת-ספרתיים, ו-score pills 3-up במסך צר — להתאים.
**מאמץ:** M.

### 2.2 ביצועים
- `listPacks`/`listClasses` הם N+1 (readdir + readFile + Zod parse לכל קובץ) בכל קריאה, בזמן שהמסך צריך 7 שדות. → אינדקס קליל (טבעי עם DB אמיתי — שאילתה אחת).
- אין `Cache-Control` — דפי ערכה מחושבים מחדש כל בקשה. → `private, max-age` על קריאות ערכה.
- CSP מתיר `'unsafe-inline' 'unsafe-eval'` — להדק עם nonces.
**מאמץ:** M.

### 2.3 מדידה, ניטור ואמינות
- ניטור שגיאות (Sentry) + request IDs + לוגים מובנים.
- אנליטיקס בסיסי (מה נכנס, מה מומר, היכן נוטשים).
- Health checks ו-alerting.
**מאמץ:** M.

### 2.4 סגירת פערי בדיקות
- טסט authz: משתמש A לא קורא/מוחק/מתרגל מול ערכת משתמש B.
- טסט persistence "בין אינסטנסים" — בדיוק הבאג ש-P0.1 מתקן ולא נתפס בטסט שחולק תהליך.
- טסט prompt-injection נגד OOXML זדוני.
**מאמץ:** M.

## P3 — צמיחת מוצר (טווח ארוך)

- **Onboarding אמיתי** — מסך פתיחה, דוגמה מודרכת, "העלה קורס ראשון".
- **מודל תשלום ומכסות** — from spend-cap → tiers (חינם מוגבל / פרימיום). קריטי כי כל ייבוא הוא עלות מודל אמיתית.
- **מצב כיתה מלא** — שיתוף ערכות, הצטרפות בקוד, הרשאות, ניהול חברים (יש כבר תשתית חלקית + TODO בקוד).
- **PWA / אופליין** — התקנה במובייל, כרטיסיות ותכנית זמינות בלי רשת.
- **אנליטיקת חזרה מרווחת** — סטטיסטיקות זיכרון, streak אמיתי, תזכורות.
- **מודרציה לתוכן משותף** — ערכות שנוצרות מקלט משתמש ומוצגות לאחרים דורשות סינון.
- **מנוע תזכורות/דיוג'סט** — מייל "מה ללמוד היום" (אפשר למנף scheduled tasks).
- **SEO ושיתופיות** — עמודי נחיתה, meta, שיתוף ערכה בקישור.
- **רב-לשוניות** — הבסיס כבר תומך (RTL/LTR, `outputLanguage`).

## נקודות אור (מה כבר טוב)

ולידציית Zod על כל ה-API · "forced tool call" ל-LLM · חסמי גודל להעלאות לפני buffering · הגנת path-traversal · security headers · 20 קבצי בדיקות · `<Ltr>`/`bdi` לבידוד נוסחאות (מדויק) · שימוש עקבי ב-`ms-*`/`me-*` ל-RTL · טיפול ב-`prefers-reduced-motion` · skip-link · חצי-ניווט מודעי-כיוון במצגת · הפרדת client/server נקייה (אין ספריות כבדות ב-client).
