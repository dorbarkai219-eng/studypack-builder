# StudyPack — מערכת העיצוב (Neo-Brutalist)

מסמך ייחוס יחיד לסגנון החזותי של האתר. כל רכיב חדש צריך להשתמש ב-tokens וב-primitives כאן — לא בערכים קשיחים. המקור: `src/app/globals.css`.

השפה החזותית: רקע קרם חמים, גבולות שחורים (ink) עבים, צללים קשיחים (בלי blur) בהיסט קבוע, פינות מעוגלות נדיבות, ומבטאים בהירים בכחול/כתום/ליים.

---

## Tokens (צבעים)

מוגדרים ב-`@theme` ונקראים דרך utilities של Tailwind (`text-*`, `bg-*`, `border-*`). כולם מתהפכים אוטומטית ב-dark mode.

| Token | תפקיד | Light |
|---|---|---|
| `canvas` | רקע הדף | `#f1eee4` (קרם) |
| `paper` | רקע כרטיסים | `#ffffff` |
| `ink` | גבולות, צללים, טקסט גוף | `#16150f` |
| `muted` | טקסט משני | `#57564a` (הושחר לניגודיות AA) |
| `lines` | קווי הפרדה עדינים | `#dcd7c8` |
| `navy` | כותרות, primary, מספרים | `#2743e6` (כחול) |
| `orange` | מבטא חם | `#f0822b` |
| `lime` | מבטא פעיל/הצלחה | `#c2ec3a` |
| `keyidea/tip/mistake/example` (+`-bg`) | קאלאאוטים | כחול/ירוק/אדום/ענבר |

**כללי צבע:**
- טקסט על רקע צבעוני = הצל הכהה ביותר מאותה משפחה, לא שחור גנרי.
- **ליים הוא מבטא לרקע כהה בלבד** — לעולם לא `text-lime` על קרם/לבן (נכשל בניגודיות).
- טקסט על כתום = `ink` (לא לבן — לבן על כתום נכשל).
- אין hex קשיח ברכיבים. משתמשים ב-tokens כדי ש-dark mode יעבוד.

## Primitives (מחלקות לשימוש חוזר)

| Class | מתי | הערות |
|---|---|---|
| `.nb-card` | כרטיס ראשי | גבול 2px ink, radius 22px, צל `5px 6px 0` |
| `.nb-card-sm` | קופסה פנימית | radius 16px, צל קטן יותר |
| `.nb-hover` | לצד `nb-card` לכרטיס אינטראקטיבי | לחיצה/הרמה שמעמיקה את הצל |
| `.nb-hero-dark` | לוח כהה בולט ("מה נשאר להיום") | ink כהה, טקסט קרם |
| `.nb-pill` (+`-lime`/`-blue`) | תגית/צ'יפ | rounded-full מתוחם |
| `.nb-btn` (+`-primary`/`-lime`/`-orange`/`-ink`) | כפתור/קישור פעולה | צל קשיח שקורס בלחיצה |
| `.nb-numeral` | מספר ספירה ענק | לזווג עם `text-6xl` וצבע `text-navy`/`text-orange` |
| `.nb-progress` (+`-orange`/`-lime`) | פס התקדמות | מבנה: `<div class="nb-progress"><span style={{width}}/></div>`; לשמור aria |
| `.nb-timer` | ספרות ספירה (mono) | ink רקע, ליים טקסט |

`.nb-btn-primary`/`.nb-pill-blue` משתמשים ב-`#2743e6` קבוע (לא ב-token) כדי שטקסט לבן יישאר קריא גם ב-dark mode.

## סקאלת radius / shadow

- **Radius:** כרטיס גדול 22–26px · קופסה 16px · כפתור 13px · pill מלא. להימנע מ-`rounded-md` (6px) שלא תואם לסקאלה.
- **Shadow:** תמיד צל קשיח `Xpx Ypx 0 0 var(--color-ink)` (בלי blur). לא להשתמש ב-`shadow-sm/lg` של Tailwind.
- **Focus:** outline שחור (`--color-ink`) 2px, מטופל גלובלית ב-globals.css.

---

## Dark mode — חוזה

- כל ה-tokens מוגדרים גם ב-`[data-theme="dark"]` וגם ב-`prefers-color-scheme: dark` (שני הבלוקים מסונכרנים). מכיוון ש-primitives קוראים `var(--color-*)`, הם מתהפכים אוטומטית.
- ב-dark, `ink` הופך בהיר — לכן גבולות וצללים נעשים בהירים על רקע כהה (זה מכוון לנאו-ברוטליזם).
- מבחן מהיר לכל רכיב חדש: אם הרקע היה כמעט-שחור, האם כל טקסט עדיין קריא? אם לא — להשתמש ב-token, לא ב-hex.

---

## חריג מוצהר: רכיבי דף הנוסחאות להדפסה

הרכיבים `TopicBox`, `FormulaBox`, `Callout`, `TermKeyStrip`, `Summaries` ו-`.cs-box` **נשארים בכוונה** בסגנון צפוף (`rounded-lg`, `border-lines`, בלי צללים) לצורך נאמנות הדפסה (A4, ריבוי עמודות). אין להעביר אותם ל-`.nb-*` — הצללים והפינות הגדולות פוגעים בהדפסה. תיעוד זה קיים כדי שלא "יתוקנו" בטעות.

במובייל (`max-width: 640px`) דף הנוסחאות עובר לעמודה אחת עם פונט קריא ועמוד ברוחב מלא — ראו את בלוק ה-`@media` ב-globals.css.

חריג מכוון (הדפסה): רכיבי ה-cheatsheet לעיל.

נשאר לשיפור עתידי (מובייל): כיווץ ה-toolbar של המצגת (~11 כפתורים) ותפריט "עוד" בנאבגשן במסכים צרים.
